# L0 kernel v1.0 设计 DESIGN

## 设计原则

L0 kernel 将基础能力拆成小包，避免跨包循环依赖和业务层反向引用。每个包优先使用标准库类型，公开面只承诺文档中列出的行为。

## 模块结构

- 错误、健康、版本和契约包提供 JSON 兼容结构，便于外部系统验证。
- 重试和时间包只提供策略与抽象，不隐式启动后台任务。
- 生命周期、同步和校验包只管理本地进程内状态，不持久化外部资源。
- 观测脱敏默认保护秘密值，显式转换仍需要调用方承担上下文判断。

## 门禁设计

`make release-preflight VERSION=v0.1.0` 串联清洁工作区检查、格式化、vet、单测、race、边界、契约、文档、示例和发布证据检查。`scripts/ci/artifact-check.sh` 额外固定 `docs/goal.md` 要求的精确工件路径，避免治理文档漂移。
