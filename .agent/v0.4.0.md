## 1. 最终架构裁决

### DEC-20260602-001：kernel 作为 x.go L0 Foundation

`kernel` 必须作为 `x.go` 的 L0 基础。

但整合方式不是合并仓库，也不是复制代码，而是：

```text
kernel = L0 工程内核
xlib-standard = 基础库模板标准
redisx / kafkax / postgresx / taosx / ossx / clickhousex / configx = L1 基础设施库
x.go = L2/L3 集成调用方与业务系统
```

### 1.1 强制原则

```text
x.go 可以依赖 kernel
L1 基础库默认依赖 kernel
xlib-standard 模板默认支持 kernel 接入
kernel 不允许依赖 x.go
kernel 不允许依赖 Redis/Kafka/PostgreSQL/TDengine/OSS/ClickHouse
kernel 不允许依赖 Prometheus/Otel SDK
kernel 不允许出现 Binance/Market/Macro/Regime/Strategy/Order 等业务概念
kernel 只提供稳定工程原语和契约
```

## 6. Spec

### REQ-001：工具链版本 SSOT

`kernel` 必须新增 `.github/versions.env`，记录最低 Go 版本、x.go 集成 Go 版本、工具版本。

### REQ-002：强 release toolchain gate

`make release-final-check` 必须阻断工具缺失、版本不一致、`GOWORK` 污染。

### REQ-003：零第三方运行时依赖

`make boundary-check` 必须证明除 Go 标准库和本模块外无外部依赖。

### REQ-004：Public API Snapshot

所有导出 API 必须生成 snapshot，API drift 必须被 CI 检测。

### REQ-005：API Compatibility Policy

删除导出 API、修改签名、删除 JSON 字段必须有 ADR、migration note、version decision。

### REQ-006：Package Maturity Map

每个 package 必须标注成熟度和 x.go 迁移风险。

### REQ-007：x.go Consumer Compatibility

kernel 发布前必须证明 x.go 可以以版本化 module tag 消费 kernel。

### REQ-008：增强 Release Manifest

Manifest 必须包含 schema version、API snapshot hash、verified Go versions、consumer evidence。

### REQ-009：Golden Behavior Contracts

核心行为必须有 golden contract，阻止行为漂移。

### REQ-010：x.go 侧 Matrix 与 Gate

x.go 必须新增 kernel baseline policy、cross-repo dependency matrix、kernel-compat-check。

### REQ-011：xlib-standard 模板同步

xlib-standard 必须支持新基础库默认接入 kernel。

### REQ-012：禁止 local replace

任何正式 release 和 main 分支禁止：

```go
replace github.com/ZoneCNH/kernel => ../kernel
```

### REQ-013：Breaking Change 必须 ADR

kernel 任何 breaking change 必须生成 ADR 并更新 release notes。

### REQ-014：Version Strategy

v0.2.0 达到 x.go foundation candidate；v1.0.0 才进入 API freeze。

### REQ-015：Evidence Protocol

所有 Issue 完成必须以 “DONE with evidence:” 结束。

### `scripts/ci/toolchain-check.sh` 模板

```bash
#!/usr/bin/env bash
set -euo pipefail

ROOT="${1:-.}"
MODE="${MODE:-local}"
VERSIONS_FILE="$ROOT/.github/versions.env"
ERRORS=0

fail() {
  echo "ERROR: $*"
  ERRORS=$((ERRORS + 1))
}

require_file() {
  local file="$1"
  if [[ ! -f "$ROOT/$file" ]]; then
    fail "missing required file: $file"
  fi
}

require_cmd() {
  local cmd="$1"
  if ! command -v "$cmd" >/dev/null 2>&1; then
    fail "missing required command: $cmd"
  fi
}

require_file ".github/versions.env"
# shellcheck disable=SC1090
source "$VERSIONS_FILE"

require_file "go.mod"

go_line="$(awk '/^go / {print $2; exit}' "$ROOT/go.mod")"
if [[ "$go_line" != "$GO_MIN_VERSION" ]]; then
  fail "go.mod go version mismatch: expected=$GO_MIN_VERSION actual=$go_line"
fi

require_cmd go

actual_go="$(go version | awk '{print $3}' | sed 's/^go//')"
case "$actual_go" in
  "$GO_MIN_VERSION"|"$GO_INTEGRATION_VERSION") ;;
  *)
    if [[ "$MODE" == "release" ]]; then
      fail "go version must be GO_MIN_VERSION or GO_INTEGRATION_VERSION: actual=$actual_go"
    else
      echo "WARN: go version not in expected set: actual=$actual_go expected=$GO_MIN_VERSION/$GO_INTEGRATION_VERSION"
    fi
    ;;
esac

gowork="$(go env GOWORK 2>/dev/null || true)"
if [[ -n "$gowork" && "$gowork" != "off" ]]; then
  fail "GOWORK must be off for standalone kernel validation: GOWORK=$gowork"
fi

if grep -R "@latest" "$ROOT/Makefile" "$ROOT/scripts" "$ROOT/.github" 2>/dev/null \
  --exclude-dir=.git --exclude='toolchain-check.sh'; then
  fail "forbidden @latest found"
fi

if [[ "$MODE" == "release" ]]; then
  require_cmd golangci-lint
  require_cmd govulncheck

  lint_version="$(golangci-lint version 2>/dev/null | head -1 || true)"
  if ! printf '%s\n' "$lint_version" | grep -q "$GOLANGCI_LINT_VERSION"; then
    fail "golangci-lint version mismatch: expected=$GOLANGCI_LINT_VERSION actual=$lint_version"
  fi

  vuln_version="$(govulncheck -version 2>/dev/null || true)"
  if ! printf '%s\n' "$vuln_version" | grep -q "${GOVULNCHECK_VERSION#v}"; then
    fail "govulncheck version mismatch: expected=$GOVULNCHECK_VERSION actual=$vuln_version"
  fi
fi

if [[ "$ERRORS" -gt 0 ]]; then
  echo "toolchain-check failed with $ERRORS error(s)"
  exit 1
fi

echo "toolchain-check passed"
```

```markdown
# kernel API Compatibility Policy

## 1. Scope

This policy governs exported Go APIs, JSON contracts, golden examples, and documented behavior.

## 2. Compatibility Levels

| Level | Meaning |
|---|---|
| experimental | May change in v0.x |
| beta | May change with migration note |
| stable-candidate | No breaking changes without ADR |
| stable | No breaking changes except major version |

## 3. Breaking Changes

The following are breaking changes:

- Removing exported function/type/const/var
- Changing exported function signature
- Removing exported struct field
- Removing JSON field
- Changing JSON field type
- Changing documented behavior
- Changing error kind semantics
- Changing health aggregation semantics
- Changing retry delay semantics without version decision

## 4. Allowed Non-breaking Changes

- Adding new exported API
- Adding optional JSON field
- Adding new error kind
- Adding new health metadata field
- Adding new no-op-compatible interface method only before v1.0

## 5. Required Process

Breaking changes require:

1. ADR
2. Migration note
3. Version decision
4. public_api.snapshot update
5. release notes
6. consumer compatibility check

## 6. v0.x Rule

v0.x may include breaking changes, but not silently.

## 7. v1.x Rule

After v1.0, breaking changes require major version.
```

#!/usr/bin/env bash
set -euo pipefail

ROOT="${1:-.}"
cd "$ROOT"

ERRORS=0

fail() {
  echo "ERROR: $*"
  ERRORS=$((ERRORS + 1))
}

if ! go list -m github.com/ZoneCNH/kernel >/tmp/kernel-module.txt 2>/dev/null; then
  fail "github.com/ZoneCNH/kernel is not required in go.mod"
else
  cat /tmp/kernel-module.txt
fi

if grep -nE '^replace[[:space:]]+github.com/ZoneCNH/kernel[[:space:]]+=>' go.mod; then
  fail "local replace for kernel is forbidden in x.go main/release"
fi

kernel_version="$(awk '$1=="github.com/ZoneCNH/kernel"{print $2}' /tmp/kernel-module.txt || true)"
if [[ -z "$kernel_version" ]]; then
  fail "kernel version not found"
fi

if [[ "$kernel_version" == v0.0.0-* ]]; then
  fail "kernel pseudo version is forbidden: $kernel_version"
fi

if [[ -f ".swarm/specs/governance/CROSS_REPO_DEPENDENCY_MATRIX.md" ]]; then
  if ! grep -q "github.com/ZoneCNH/kernel" .swarm/specs/governance/CROSS_REPO_DEPENDENCY_MATRIX.md; then
    fail "kernel not registered in cross repo dependency matrix"
  fi
else
  fail "missing CROSS_REPO_DEPENDENCY_MATRIX.md"
fi

go test ./internal/integration/kernelcompat/...

if [[ "$ERRORS" -gt 0 ]]; then
  echo "kernel-compat-check failed with $ERRORS error(s)"
  exit 1
fi

echo "kernel-compat-check passed"
